About Me

Student in the Advanced Programming course, learning Python, Flask, Git and web development. This portfolio page presents my Online Library application and how it was developed.

Online Library – Overview

The Online Library is a small web application that lets users manage a collection of media items such as books and movies. Users can add new items, search by title or category, update existing entries and delete items from the library.

The project follows a simple client–server architecture: a Flask backend exposes REST endpoints and a web frontend (HTML, CSS and JavaScript) communicates with the backend using HTTP requests.

Main features

  • Add media items with title, author, year, category and type.
  • Search and filter items using a search box and category selection.
  • Edit or delete existing items through the interface.
  • Data is stored on the server side in a separate storage module.

Technologies and tools

  • Python & Flask – implement the web API for media management.
  • HTML, CSS, JavaScript – build the user interface for the online library.
  • Git, GitHub, SourceTree – version control and remote repository hosting.
  • Visual Studio Code – main development environment.

How to run the project

  1. Clone the repository from GitHub: git clone https://github.com/VinuSre/OnlineLibrary.git
  2. Open the project folder in Visual Studio Code.
  3. Install required Python packages (for example Flask) in a virtual environment.
  4. Start the backend server by running backend.py.
  5. Open the index.html page in a browser to access the Online Library interface.

While developing, the application runs on http://127.0.0.1:5000/. For other devices on the same network, the server can be started with host 0.0.0.0 and accessed via the laptop’s local IP address.

What I learned

  • Designing and implementing a REST-style backend in Flask.
  • Connecting a JavaScript frontend to a Python API using HTTP requests.
  • Using Git and GitHub for version control, including commits, branches and pushes.
  • Debugging common issues such as CORS errors, wrong routes or missing dependencies.
